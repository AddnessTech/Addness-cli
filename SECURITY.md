# Security policy

## Supported versions

Security fixes are provided for the latest released version of Addness CLI.

## Reporting a vulnerability

Do not report vulnerabilities through public GitHub issues.

Use GitHub private vulnerability reporting or GitHub Security Advisories for
this repository. Include:

- Affected version or commit.
- Reproduction steps.
- Impact assessment.
- Whether any credential, token, or user data may be exposed.

Maintainers will acknowledge reports as soon as practical, triage impact, and
coordinate disclosure after a fix is available.

## Secret handling

Never commit API keys, tokens, private keys, local credential files, `.env`
files, or customer data. If a secret is committed, rotate it immediately and
notify maintainers privately.
